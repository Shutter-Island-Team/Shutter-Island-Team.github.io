Animal behavior

Boids simulation

The ecosystem of this project use the boid technique. A boid is an autonomous agent who is aware of its environment and move considering the steering force applied to it. Here is an example of boids which are moving as a group.

Simple boid system. This simulation is computed live on your browser.

To improve the behavior of the boids, we used an automaton which indicate the transitions between the different states of the boid. We also used some state variables which indicate if a boid is hungry, thirsty or tired. The following automaton represent a simplified version of the automaton used in this project.

Boid state automaton

Boid choice

In order to populate the world, two type of boids were used

  • Some living boid which are mobile : The wolf and the rabbit
  • Some immobile boids : The tree and the carrot.

Trees and carrots were considered here as boid to handle the relations between them and the others moving boids. We could therefore for example use a delimitation zone to avoid the collision between the boids and the environmental elements. It is the same rule which indicate why two animals do not intersect and why a living boid do not touch an environmental element.

Real time computing

One concern of this project was to make it run in real time. Here are some optimisations we made to improve the speed of the program.

  • Parallel computing of the positions of the boids
  • Sending simultaneously multiple data to the GPU.
  • Division of the map into a grid to restrict the number of interact between boids on each other.

In the end, it was possible to run the program with around 1000 boids on a standard machine at 60 fps.

Environment-Animal interactions

The boids also interact with the environment. For example :

  • The boid's position is at the corresponding height of the map.
  • The boids remember where the lakes are and are careful not to drown in them.
  • The boids do not venture over the oceans.
